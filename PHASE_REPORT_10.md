# Phase 10 Report — Trust & Safety, Age Gating, Consent, Wellness

Integrates the safety/compliance feature set (from the uploaded
`gwaveaiALLfeatures` build) into the full Phase 1–9 platform. The upload
was based on an older Phase-2 snapshot, so nothing was copied wholesale:
each feature was reconciled with the existing schema (Phase 8 already had
reports/moderation) and re-validated. All gates pass; the migration chain
is now `0001 → 0010`.

## 1. Migration `20260707120000_safety_wellness.sql`

### User blocks
- `blocks (blocker_id, blocked_id)` self-managed via RLS;
  `is_blocked(a, b)` SECURITY DEFINER helper (mutual, either direction).
- `can_view_post_id()` reworked: posts from a blocked/blocking user are
  invisible in both directions (feed, profile, comments, reactions and
  shares all flow through this function).
- Comments select policy reworked the same way.

### Soft removal (moderation)
- `posts.removed_at` / `comments.removed_at`; removed content is hidden
  from everyone except its author and moderators.
- `resolveReport(…, "remove")` now soft-removes instead of hard-deleting
  (reversible + auditable). Profile reports resolve to a 7-day suspension.

### Profile reports
- `reports.profile_id` added; `report_single_target` constraint now
  enforces exactly one of post/comment/profile.
- Admin moderation queue shows the reported user with a suspension hint.

### Age safety
- `profiles.birth_date` (checked range), `age_band` enum, and
  `age_band_of()` / `is_adult()` / `is_minor()` / `my_age_band()` helpers.
- App-side gate `requireAdult()` on `/strains/*` and `/farm/*`
  (cannabis-related) — minors land on `/restricted`, missing DOB goes to
  onboarding. Nav hides adult-only entries via `visibleNav()`.
  DB read policies stay open for the API-key-scoped B2B API.

### Consent & right to erasure
- Append-only `consents` (terms/privacy versions, optional guardian
  fields) + latest-version columns on `profiles`; captured at onboarding
  together with the new required DOB + terms checkbox
  (`src/lib/consent.ts` holds the policy versions).
- `deletion_requests` — request/cancel own; processed by an operator.

### Wellness library
- `wellness_items` (`dhamma` / `meditation` / `radio` / `health`), public
  read, admin write; starter content in `supabase/seed/wellness_seed.sql`.

## 2. Application features

| Route / component | What it does |
| --- | --- |
| `/wellness` | Dhamma reflections, meditation timer (SVG progress ring + Web-Audio chime), listening links, health guidance with a clear not-medical-advice notice |
| `/learn` | Age-banded learning tracks (STEM for kids, coding for teens, agri-science for adults) |
| `/restricted` | Friendly 18+ wall pointing minors to /learn |
| `/settings` | Consent status, JSON data export (right to access), request/cancel account deletion — the profile-menu Settings link now resolves |
| `/tools/qr` | QR generator (text/URL, Wi-Fi, vCard, PromptPay) with PNG/SVG download, color + error-correction controls; registered in the tools hub |
| Post menu | Report (existing) + **Block user** |
| Profile page | New ⋯ menu: report user, block/unblock |
| Onboarding | DOB (required) + Terms/Privacy acceptance, recorded to `consents` |

## 3. i18n — four locales

- `th` (ไทย) and `zh` (中文) added alongside `en`/`my`;
  `LOCALE_LABELS` drives the switcher.
- New catalogs were generated by deep-merging the uploaded th/zh
  translations over the full English catalog, so every key in all 30
  namespaces resolves (untranslated strings fall back to English rather
  than crashing next-intl).

## Notes / follow-ups

- Real Terms of Service & Privacy Policy documents (and legal review) are
  still required — `src/lib/consent.ts` versions gate re-acceptance.
- Deletion requests are queued for an operator; automated cascade
  (storage, backups) is a deliberate manual step.
- Guardian-consent workflow for minors is captured in the schema but not
  yet built into the UI.
- Blocking hides posts/comments; messenger blocking is a follow-up (the
  launch checklist tracks it).
