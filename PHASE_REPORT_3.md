# Phase 3 ‚Äî Knowledge Search (Strains + Minerals)

Adds the Leafly-style cannabis strain database and a minerals/metals
reference database, both publicly searchable, wired into a new global
search dropdown in the navbar.

## What was built

### Data layer

- **Migration** `supabase/migrations/20260705140000_knowledge_search.sql`:
  - `pg_trgm` extension and a `strain_type` enum (indica/sativa/hybrid).
  - `strains` table: name, slug, type, THC/CBD %, `effects[]`,
    `flavors[]`, `terpenes[]`, grow difficulty, flowering weeks,
    indoor/outdoor yields, description ‚Äî plus a **generated `tsvector`
    column** (name weighted A, description B) with a GIN index and a
    **GIN trigram index on name** for fuzzy matching. Additional indexes
    on type, THC and effects (GIN) for the browse filters.
  - `minerals` table: name, slug, symbol/formula, category, Mohs
    hardness, density, `properties jsonb`, `uses[]`, description ‚Äî with
    the same tsvector + trigram indexing.
  - **RLS**: `select` granted to `anon` and `authenticated` (searching
    requires no login); all writes require `is_admin()` (a SECURITY
    DEFINER helper checking `profiles.role in ('admin','super_admin')`).

- **Seed** `supabase/seed/knowledge_seed.sql` ‚Äî generated by
  `scripts/generate-knowledge-seed.mjs` (committed; edit the script and
  re-run to regenerate):
  - **207 well-known cannabis strains** (OG Kush, Blue Dream, Sour
    Diesel, ‚Ä¶ including 10 CBD-rich cultivars like Harlequin and ACDC
    with realistic high-CBD/low-THC numbers). Attributes are derived
    deterministically from the strain name so re-runs are stable.
  - **100 minerals/metals** with curated real-world data (formula,
    category, Mohs hardness, density, uses ‚Äî e.g. Quartz SiO‚ÇÇ 7 / 2.65,
    Gold Au 19.3, Tungsten W 19.25 with its 3422¬∞C melting point noted).

- **Query layer** `src/lib/db/knowledge.ts`: paginated `listStrains`
  (name query + type + min-THC + effects filters), `listMinerals`
  (query + category), slug getters, category list, and
  `quickSearchKnowledge` for the dropdown. LIKE wildcards escaped.

### Features

1. **Global search (navbar)** ‚Äî `GlobalSearch` replaces the plain
   input: debounced (250 ms) requests to `GET /api/search`, results
   grouped into People / Posts / Strains / Minerals, fully keyboard
   navigable (‚Üë/‚Üì/Enter/Escape), stale-response guard, "See all
   results" footer to the full `/search` page. Strains/minerals appear
   even for logged-out visitors; people/posts require a session.
2. **`/strains`** ‚Äî Leafly-style browse grid with filters: type tabs
   (all/indica/sativa/hybrid), minimum-THC chips, multi-select effect
   chips, name search box, prev/next pagination. Strain cards show a
   colored type badge, THC/CBD and top effects.
3. **`/strains/[slug]`** ‚Äî detail page with THC/CBD meters, effect
   bars, flavor and terpene chips, and a growing-info card (difficulty,
   flowering weeks, indoor/outdoor yields).
4. **`/minerals`** ‚Äî browse grid with category chips (11 categories)
   and name search; cards show formula, hardness and density.
5. **`/minerals/[slug]`** ‚Äî detail page with a property table (Mohs,
   density, category + every key in the `properties` jsonb) and use
   chips.
6. Both sections are **public** (new `(knowledge)` route group with a
   navbar-only layout; middleware never guarded these paths) and are
   linked from the left sidebar. The full `/search` page now shows a
   "Knowledge base" section alongside people and posts.

## Quality gates

```bash

pnpm lint        # no warnings or errors
pnpm build       # compiled successfully
```

Migration chain (0001 ‚Üí 0004) + both seed files were applied to a
scratch PostgreSQL 16 instance with asserts: seed integrity (207
strains with sane THC/CBD/flowering ranges, 100 minerals in 11
categories), anon read access, anon and non-admin writes rejected by
RLS, admin insert allowed, and fuzzy name search returning sensible
matches.

## How to test manually

1. Apply the migration, then run the knowledge seed
   (`psql ... -f supabase/seed/knowledge_seed.sql` or via
   `supabase db reset` if wired into your seed flow).
2. While **logged out**, open `/strains` and `/minerals` ‚Äî both should
   render and filter without redirecting to login.
3. Type "blue" in the navbar search ‚Äî the dropdown should list Blue
   Dream/Blueberry/‚Ä¶ under Strains (plus people/posts when logged in);
   arrow keys + Enter should navigate.
4. On `/strains`: filter by Sativa + THC 20%+ + "energetic"; open a
   strain and check effect bars, terpene chips and the growing card.
5. On `/minerals`: filter by "Sulfide", open Galena and check the
   property table; search "gold".

## Notes / follow-ups

- Strain attributes in the seed are deterministic estimates for demo
  purposes; real lab data can replace them row-by-row (admin-only
  writes are already enforced).
- The tsvector columns are indexed and ready for ranked `websearch`
  queries later; current queries use trigram-friendly ILIKE which fits
  the "type a name" use case.
- Strain/mineral images (`image_url`) are supported by the schema but
  not seeded.
- Membership & payments are Phase 4.
